## 4. UI/UX

### 4.1. Основные экраны

#### 4.1.1. Карта уровней (Level Map)
**Элементы:**
- Визуальное представление эпизодов (острова, главы, зоны)
- Иконки уровней с индикацией:
  - Пройден (количество звёзд)
  - Доступен для игры
  - Заблокирован
  - Hard Level (специальная пометка)
- Прогресс игрока (текущий уровень)
- Кнопки навигации между эпизодами

**Взаимодействие с игровой логикой:**
- Callbacks: OnLevelSelected(levelNumber)
- Проверка доступности: IsLevelUnlocked(levelNumber)
- Отображение достижений: GetStarsForLevel(levelNumber)

#### 4.1.2. Игровое поле (Game Board)
**Элементы:**
- Grid с игровыми элементами
- Счётчик ходов (оставшихся)
- Отображение целей уровня с прогрессом
- Счётчик очков
- Индикатор прогресса для целевых очков (если применимо)
- Кнопка паузы
- Кнопка бустеров (опционально, если есть pre-game бустеры)

**Обновление состояния:**
- Events: OnMoveMade, OnScoreChanged, OnObjectiveProgress
- Real-time обновление счётчиков

#### 4.1.3. Экран результатов (Level Complete/Failed)
**Победа:**
- Количество набранных звёзд
- Финальный счёт
- Распределение очков (бонусы, комбо и т.д.)
- Кнопки: "Next Level", "Replay", "Back to Map"

**Поражение:**
- Причина поражения (нет ходов / цели не достигнуты)
- Прогресс по целям
- Кнопки: "Retry", "Back to Map", "Buy Extra Moves" (опционально)

---

### 4.2. Навигация

#### 4.2.1. Flow между экранами
```
Main Menu
    ↓
Level Map
    ↓
Level Start (pre-game экран с целями и бустерами)
    ↓
Game Board
    ↓
Level Complete/Failed
    ↓
Level Map (или Next Level)
```

#### 4.2.2. Информационная архитектура
- Минимум кликов до начала игры
- Возможность быстрого возврата на карту из любого экрана
- Автосохранение прогресса
